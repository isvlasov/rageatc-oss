# Edge Cases and Common Pitfalls

This file provides guidance on handling unusual verification scenarios and avoiding common fact-checking mistakes.

## Additional Edge Cases

### Time-Constrained Verification

**Challenge:** Limited time available for thorough verification whilst maintaining standards.

**Adapted Protocol:**

**Priority 1 (Always verify):**
- High-significance claims affecting key conclusions
- High-risk claims with potential for harm if wrong
- Claims contradicting established knowledge
- Statistics and numbers central to argument

**Priority 2 (Verify if time permits):**
- Supporting details that don't affect main conclusions
- Background claims with low stakes
- Claims with low confidence already noted

**Priority 3 (Document as unverified):**
- Tangential claims not central to argument
- Low-risk claims where error has minimal impact
- Claims clearly marked as speculative

**Workflow adjustments:**
1. **Use tertiary sources for initial orientation** - Wikipedia, encyclopaedias for quick context
2. **Focus on primary sources for key claims** - Go straight to authoritative sources for Priority 1
3. **Accept fewer triangulation sources** - 2 sources instead of 3 if both high-quality Tier 1
4. **Document time constraints explicitly** - Note in limitations section
5. **Lower confidence ratings appropriately** - Use "Possible" instead of "Likely" when triangulation incomplete

**Example limitation statement:**
"Verification conducted under time constraints. High-priority claims verified with primary sources; lower-priority claims verified with 2 sources instead of standard 3. Confidence ratings adjusted accordingly."

### Paywalled or Inaccessible Primary Sources

**Challenge:** Best source is behind paywall, in archive, or otherwise inaccessible.

**Resolution strategies:**

**Strategy 1: Use institutional access**
- University library access
- Public library databases
- Professional organisation memberships
- Open access repositories

**Strategy 2: Rely on high-quality secondary sources**
- Cite reputable secondary sources that accessed primary source
- Verify multiple secondary sources agree on what primary source says
- Note in documentation: "Primary source inaccessible; verified via [secondary sources]"

**Strategy 3: Use source abstracts or summaries**
- Academic abstracts often freely available
- Government report executive summaries
- Official press releases about research
- Verify these are from original source, not third-party interpretation

**Strategy 4: Contact authors or organisations**
- Request copy of paper from author (often freely provided)
- Contact organisation for access
- Check if pre-print version available

**What NOT to do:**
- Don't cite sources you haven't read or verified through reliable intermediaries
- Don't assume secondary interpretations are accurate without corroboration
- Don't inflate confidence levels when relying on secondary sources for inaccessible primaries

### Claims About Ongoing or Future Events

**Challenge:** Verifying claims about situations still unfolding or predictions.

**Handling protocols:**

**For ongoing events:**
- Note date and time of verification explicitly
- Use language acknowledging fluidity: "As of [date/time], X"
- Flag that situation may have changed
- Note most recent authoritative update
- Lower confidence for fast-changing situations

**Example:**
"As of 16 January 2026, 14:00 GMT, [source] reports X. Situation is ongoing; verify current status before relying on this information."

**For predictions:**
- Categorise as "Uncheckable" (cannot verify until future)
- Can verify track record of predictor
- Can verify factual basis underlying prediction
- Never assign "Confirmed" to predictions
- Maximum confidence: "Possible" or "Likely" based on quality of reasoning and evidence

**Example:**
"Prediction cannot be verified until 2030. Factual basis (current trends, demographics) is confirmed. Predictor has mixed track record (60% accuracy on similar predictions). Categorised as uncheckable prediction with plausible factual foundation."

### Claims Involving Classified or Confidential Information

**Challenge:** Claims about information not publicly verifiable.

**Handling:**

**If sources exist but are classified:**
- Note claim is unverifiable with public sources
- Document why verification impossible
- Note any partial corroboration from declassified or leaked sources
- Confidence: Unverified (not disputed, just unverifiable)

**If claim comes from whistleblower or leak:**
- Assess credibility of source (track record, corroboration)
- Look for indirect verification (public consequences of claimed secret actions)
- Note inherent verification limitations
- Use cautious confidence language

**If claim is speculation about classified information:**
- Categorise as unverifiable
- Do not attempt verification
- Note speculative nature

### Mutually Exclusive Claims Presented as Compatible

**Challenge:** Source presents contradictory claims without acknowledging conflict.

**Example:** "Crime is rising rapidly, and we've never been safer."

**Handling:**
1. **Identify the contradiction** - Note specific incompatibility
2. **Check definitions** - Are different metrics being used? (reported crime vs victimisation surveys)
3. **Check timeframes** - Are claims about different time periods?
4. **Verify each claim separately** - Both, one, or neither may be supported
5. **Note the contradiction explicitly** - "Source presents contradictory claims: X and Y. These cannot both be true under standard definitions."
6. **Resolve if possible** - "Apparent contradiction resolves if X refers to [context A] and Y refers to [context B]"

### Statistical Claims Without Error Margins

**Challenge:** Statistics presented without confidence intervals, margins of error, or significance testing.

**Handling:**

**For survey data:**
- Check if margin of error is standard for sample size
- Calculate approximate margin if sample size known
- Note absence of error margin in limitations
- Reduce confidence if precision claimed exceeds what's justified

**For small differences:**
- Flag differences within likely margin of error
- Example: "52% vs 48%" without error margin may not be meaningful difference
- Note: "Difference may not be statistically significant"

**For comparative claims:**
- Check if statistical significance testing performed
- Note if comparison lacks significance test
- Use cautious language: "appears to differ" not "differs significantly"

## Common Pitfalls to Avoid

### 1. Over-Reliance on Tertiary Sources

**Pitfall:** Using Wikipedia or encyclopaedias as final verification source.

**Why problematic:**
- Tertiary sources are compilations, not original research
- May contain errors or be outdated
- Insufficient for formal citation

**Correction:**
- Use tertiary sources for orientation only
- Always trace back to primary or secondary sources
- Verify key claims in original sources

**Example:**
- Wrong: "According to Wikipedia, X is true" (insufficient)
- Right: "According to [primary source cited by Wikipedia], X is true"

### 2. Vertical Reading Trap

**Pitfall:** Assessing sources solely by examining their own "About" pages or self-descriptions.

**Why problematic:**
- Every source presents itself positively
- Bias and credibility issues not self-reported
- Missing external perspective on reliability

**Correction:**
- Practice lateral reading: leave the source and search what others say about it
- Check fact-checking sites' assessments
- Look for independent reviews of source credibility
- Search "[source name] + bias" or "[source name] + reliability"

**Example:**
- Vertical reading: "Website says it's 'non-partisan research institute'" → Accept as unbiased
- Lateral reading: Search "Institute for X + bias" → Find multiple reports of partisan funding

### 3. False Balance

**Pitfall:** Giving equal weight to fringe positions vs mainstream expert consensus.

**Why problematic:**
- Creates impression of equal scientific standing when there isn't
- Misleads about actual state of evidence
- Elevates marginal views inappropriately

**Correction:**
- Weight by source quality and evidence strength, not by giving "both sides" equal space
- Note consensus explicitly: "97% of climate scientists agree X; 3% dispute"
- Don't present outlier views as equally credible as consensus

**Example:**
- False balance: "Some scientists say climate change is real; others disagree" (implies equal weight)
- Accurate: "Overwhelming scientific consensus (97%+ of climate scientists) confirms human-caused climate change. Small minority of scientists dispute aspects of consensus."

### 4. Confidence Inflation

**Pitfall:** Presenting "Likely" or "Possible" claims as "Confirmed" or using definitive language for uncertain claims.

**Why problematic:**
- Misrepresents strength of evidence
- Creates false certainty
- Undermines trust when claims later disputed

**Correction:**
- Use calibrated language appropriate to evidence strength
- Preserve uncertainty where it exists
- Never upgrade confidence beyond what evidence supports

**Examples:**
- Confidence inflation: "X is confirmed" (when evidence is "likely" strength)
- Calibrated: "Strong evidence suggests X, though some uncertainty remains"

### 5. Hidden Disagreements

**Pitfall:** Quietly choosing one source when authoritative sources conflict, without noting disagreement.

**Why problematic:**
- Conceals important uncertainty
- Misrepresents state of knowledge
- Readers can't assess controversy for themselves

**Correction:**
- Surface disagreements transparently with attribution
- Note why sources disagree if determinable
- Present both positions with appropriate weighting

**Example:**
- Hidden: "UK unemployment is 4.2%" (choosing Source A, ignoring Source B saying 3.8%)
- Transparent: "UK unemployment is 4.2% (ILO measure) or 3.8% (claimant count), depending on definition used"

### 6. Opinion Fact-Checking

**Pitfall:** Attempting to verify subjective values, preferences, or aesthetic judgements.

**Why problematic:**
- Opinions don't have objective truth values
- Wastes verification resources
- May misrepresent opinion as factual error

**Correction:**
- Categorise opinions correctly (Category 3)
- Don't attempt fact-checking of subjective statements
- Focus verification on factual premises underlying opinions

**Example:**
- Opinion fact-checking: Trying to verify "modern art is terrible"
- Appropriate: Verify factual premise: "modern art sales have declined" (if claim provides this factual basis)

### 7. Single-Source Acceptance

**Pitfall:** Accepting significant factual claims from one source without triangulation.

**Why problematic:**
- Single source may have errors
- Outlier findings may not be representative
- Fabrications or hallucinations go undetected

**Correction:**
- Seek independent corroboration for important assertions
- Minimum 3 independent sources for significant claims
- If only one source available, note limitation and reduce confidence

**Example:**
- Single-source: "According to [one source], X" → Confidence: Likely or Possible
- Triangulated: "According to [Source A], [Source B], and [Source C], X" → Confidence: Confirmed

### 8. Context-Free Statistics

**Pitfall:** Verifying numbers without checking definitions, methodologies, or timeframes.

**Why problematic:**
- Same metric can mean different things (unemployment: ILO vs claimant count)
- Comparing incompatible statistics
- Missing important context affecting interpretation

**Correction:**
- Always check definitions and methodologies
- Verify comparing like with like
- Note timeframes and contexts
- Check for changes in measurement standards over time

**Example:**
- Context-free: "Unemployment is 4.2%" (which measure? when? which population?)
- Contextualised: "UK unemployment is 4.2% (ILO measure, ages 16+, November 2025, seasonally adjusted)"

### 9. Hallucination Blindness

**Pitfall:** Assuming AI fluency equals accuracy; accepting plausible-sounding claims without verification.

**Why problematic:**
- AI generates fluent text regardless of accuracy
- High confidence does not mean correctness
- Hallucinations often sound authoritative

**Correction:**
- Always verify plausible-sounding claims against sources
- Especially vigilant with AI-generated content
- Check for source grounding (can claim be traced to provided context?)
- Apply extrinsic hallucination checks systematically

**Example:**
- Hallucination blindness: "AI said Marie Curie won 3 Nobel Prizes; sounds right" → Accept without checking
- Proper verification: Check Nobel Prize official records → Find only 2 Nobel Prizes → Flag hallucination

### 10. Expertise Overreach

**Pitfall:** Making definitive judgements on specialist claims beyond your domain expertise.

**Why problematic:**
- Misinterpret technical content
- Miss important caveats or context
- Provide false verification

**Correction:**
- Recognise expertise boundaries
- Flag for domain expert review when appropriate
- Note limitations of general fact-checking for specialist claims
- Verify process and methodology rather than substantive conclusions when beyond expertise

**Example:**
- Overreach: Definitively verifying complex medical treatment efficacy claim without medical expertise
- Appropriate: Verifying the claim appears in peer-reviewed medical journals; noting expert review needed for substantive evaluation

### 11. Recency Bias

**Pitfall:** Always preferring newest sources regardless of quality or context.

**Why problematic:**
- Newer doesn't always mean better (especially for historical topics)
- Seminal older research may be higher quality
- Recent sources may lack perspective or may not be peer-reviewed yet

**Correction:**
- Assess recency contextually based on field
- For fast-moving fields (AI, tech): prefer recent
- For established knowledge: older authoritative sources acceptable
- Check if newer research supersedes older, or if both remain relevant

**Example:**
- Recency bias: Rejecting 1905 Einstein paper because "too old"
- Appropriate: Using foundational older work where relevant; seeking recent work for current state of field

### 12. Circular Citation Chains

**Pitfall:** Three sources all citing the same original claim without independent verification, appearing as triangulation.

**Why problematic:**
- Appears to be corroboration but isn't
- All sources depend on single original (which may be wrong)
- False sense of consensus

**Correction:**
- Verify sources are genuinely independent
- Check if sources cite each other or common single source
- Trace back to truly independent verification

**Example:**
- Circular: Source A, B, and C all cite Source D → Only 1 independent verification, not 4
- Independent: Sources A, B, and C each conducted separate research → True triangulation

## Quick Reference

### Pre-Verification Checklist

Before accepting a claim as verified, confirm:

- [ ] Used primary or high-quality secondary sources (not just tertiary)
- [ ] Practiced lateral reading to assess source credibility
- [ ] Weighted by evidence quality, not false balance
- [ ] Used calibrated confidence language
- [ ] Surfaced any disagreements transparently
- [ ] Correctly categorised opinions vs facts
- [ ] Triangulated with 3+ independent sources for significant claims
- [ ] Checked context for statistics (definitions, timeframes, methodologies)
- [ ] Verified AI claims against sources, not just accepted fluent output
- [ ] Flagged specialist claims for expert review when beyond expertise
- [ ] Assessed recency appropriately for field
- [ ] Ensured sources are genuinely independent (no circular citations)

### Red Flags Requiring Immediate Escalation

- Specialist medical/legal claims with high stakes
- Classified information claims
- Contradictory claims from equally authoritative sources (unresolvable)
- Novel research with massive implications (extraordinary claims)
- Complex methodological disputes beyond general fact-checking
- Claims about ongoing crises requiring real-time accuracy
